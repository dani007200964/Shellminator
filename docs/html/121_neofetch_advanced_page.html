In the previous example, we saw how to get our custom Neofetch callback up and running. Now, letâ€™s take it a step further and explore how to customize it!

Currently, there are 7 built-in parameters that you can modify:

  • header_text â†’ Name displayed in the header (C-string)
  • header_line â†’ Character used for the line under the header (must be a single character)
  • fw_text â†’ Firmware text (C-string)
  • cpu_text â†’ Processor/microcontroller identifier (C-string)
  • compiler_text â†’ Compiler identifier (C-string)
  • compile_date_text â†’ Compilation date (C-string)
  • author_text â†’ Project owner (C-string)

Each of these parameters also has a color and style option. For example, header_text can be customized using header_text_color and header_text_style.

Additionally, you can format the parameter labels using field_text_color and field_text_style.

Custom User Fields

There are also 5 user-defined fields that you can fully customizeâ€”**names, values, colors, and styles**:

  • user_field_1_label â†’ Field name (C-string)
  • user_field_1_text â†’ Field value (C-string)
  • user_field_1_text_color â†’ Field text color (C-string)
  • user_field_1_text_style â†’ Field text style (C-string)
  • user_field_2_label â†’ Field name (C-string)
  • ...

One important thing to note: User fields are only displayed if their corresponding field_label is not an empty C-string.

Implementation Details
